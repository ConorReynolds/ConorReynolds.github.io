Propositional Logic in Coq

All natural deduction rules for propositional logic have been covered by now in the lectures. Let’s put them into practice.

Proofs Involving ‘And’ and ‘Or’

Recall that an elimination rule for a propositional connective ‘removes’ that connective, and more generally explains how the connective can be used or broken down into simpler parts. In Coq, it is better to think of there being two different elimination rules, which we will call left- and right-elimination. Which one you use will depend on whether the connective appears in the hypotheses (left) or in the goal (right).

For example, if we have a hypothesis of the form \(A \wedge B\), the left-elimination rule for ‘and’ says the following:

\[\begin{aligned} \frac{\Gamma, A \wedge B \vdash C}{\Gamma, A, B \vdash C} \end{aligned}\]

In plain English: if we know ‘\(A \wedge B\)’, then we know \(A\), and we also know \(B\). It breaks apart the original hypothesis so that we can use \(A\) and \(B\) separately. Not very interesting.

A more interesting example involves the hypothesis \(A \vee B\). The left-elimination rule for ‘or’ is as follows:

\[\begin{aligned} \frac{\Gamma, A \vee B \vdash C}{\Gamma, A \vdash C \qquad \Gamma, B \vdash C} \end{aligned}\]

In plain English: if we can prove \(C\) using \(A \vee B\), then in fact we should be able to prove \(C\) from \(A\) only, and we should be able to prove \(C\) from \(B\) only. The point is that if we claim that either \(A\) or \(B\) is sufficient to prove \(C\), then either one alone should be enough to prove \(C\).

The right-elimination rules explain what happens when ‘and’ or ‘or’ appear in the goal, rather than in our hypotheses. The right-elimination rule for conjunction is:

\[\begin{aligned} \frac{\Gamma \vdash A \wedge B}{\Gamma \vdash A \qquad \Gamma \vdash B} \end{aligned}\]

This says that, if we want to prove \(A \wedge B\), then we must prove \(A\) only, and then we must prove \(B\) only.

The right-elimination rules for disjunction are:

\[\begin{aligned} \frac{\Gamma \vdash A \vee B}{\Gamma \vdash A} \qquad \frac{\Gamma \vdash A \vee B}{\Gamma \vdash B} \end{aligned}\]

This gives us two ways to prove \(A \vee B\): either we prove \(A\), or we prove \(B\). Either is fine.

To apply the left-elimination rule for ‘and’ or ‘or’, use the tactic destruct. To apply the right-elimination rule for ‘and’, use the tactic split. To apply the right-elimination rules for ‘or’, use the tactics left or right, depending on which side of the disjunction we think we can prove.

Proofs Involving Negation

Recall that \(\neg A\) in constructive propositional logic is really just a shorthand for \(A \to \bot\), which means that \(A\) implies a contradiction. In Coq, the false-by-definition proposition \(\bot\) is called False. The proposition \(\bot\) is sometimes called ‘bottom’.

Since negation is shorthand for a special case of implication, you can use all the tactics from the first lab.

If you see a goal of the form ~A, you can reduce it to its definition A -> False with the tactic red—short for ‘reduce’—after which you can use intros. Alternatively, if you want to unfold all instances of negation, including in hypotheses, you can use unfold "~" in *. This can help you spot valid applications of the apply tactic.

Classical Logic

Coq’s logic is constructive. This means that the law of excluded middle is not an axiom in Coq’s logic. If you want to reason classically, you must add LEMlaw of excluded middle as an axiom or hypothesis explicitly.

Axiom LEM : forall P : Prop, P \/ ~P.
Hypothesis LEM : forall P : Prop, P \/ ~P.

When you know LEMlaw of excluded middle, you can always reason by case analysis. For example, if you write destruct (LEM A), the proof will split into two cases: one in which you know \(A\), and one in which you know \(\neg A\).

You will need LEMlaw of excluded middle for one of the final four questions on the De Morgan laws—but we’re not telling you which one. That’s something for you to figure out yourself.
